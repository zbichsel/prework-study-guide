HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The headers 1-6 define the size of the header; we only use h1 once per page.
  • The img element contain image information as well as location information.
  • The p element represents a paragraph or block of text inside the body element.
  • The ul, ol, and li elements represent unordered lists, ordered lists, and list items.
  • The br element creates a line break between two blocks of content.
  • The a element, or ancher element, creates links to the same webpage or other webpages.
  • The href attribute points to the URL for the link.

CSS

  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • Inline CSS makes a single change
  • Internal CSS contains rules for the webpage within the head element
  • External CSS creates a file with rules for the entire webpage
  • CSS allows you to stylize the webpage within the HTML

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull origin main: a way to check to see if the local branch is in sync with the base branch in GitHub
  • git push origina main: push local commits to the main branch of the origin remote
  • git checkout main: switches to main branch
  • git clone git@gitlab.com/:gitlab-tests/sample-project.git: clone git repository with SSH
